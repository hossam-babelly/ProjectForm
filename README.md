# نموذج طرح دراسة مشروع 📋

نظام ويب متكامل (Frontend + Backend) لمؤسسة **«المجد التنموية»**: يستقبل بيانات دراسة جدوى مشروع عبر فورم عربي من **11 خطوة**، ويُولّد منها تلقائياً ملفّ **Word** وملفّ **Excel** احترافيّين (RTL)، يُحمّلهما فوراً ويُرسلهما بالبريد، مع **لوحة إدارة** لعرض/تنزيل/تعديل/حذف الطلبات.

> 📘 لتفاصيل معمارية شاملة (خريطة الكود، نظام الألوان، نقاط التطوير)، انظر **`PROJECT_HANDOFF.md`**.

## ✨ المخرجات

**ملف Word (.docx)** بـ 12 صفحة RTL أفقية (A4) بخط Sakkal Majalla:
- ترويسة المؤسسة على كل صفحة (شعار أعلى + تواصل/زخرفة أسفل).
- عنوان غلاف: «قسم المشاريع التنموية» (أزرق) + «نموذج طرح دراسة مشروع» (برتقالي).
- جداول مالية بعرض الصفحة بألوان دقيقة، مع **صفّ إجمالي سنوي** أسفل جدولَي الإيرادات والتشغيلية.
- **الهيكل التنظيمي** كجدول Word أصلي ملوّن حسب نوع الموظّف (إداري أزرق · تنفيذي أخضر · مزوّد خدمة برتقالي) — ليُفتح في Word دون أي رسالة تلف.
- **صفحة خلاصة مالية** بصرية: 5 بطاقات + صافي الربح + فترة الاسترداد + حلقتا الهامش/العائد + أشرطة مقارنة.

**ملف Excel (.xlsx)** بـ 12 ورقة RTL مترابطة بمعادلات حيّة (تعديل أي مُدخل يُعيد حساب كل المخرجات)، مع تنسيق شرطي: **دوائر نِسَب** (`5Quarters`) وأشرطة بيانات في ورقة الخلاصة.

> يُبنى مستند Word بمكتبة `docx`، ثمّ يُحقَن الهيكل التنظيمي واللوحة المالية عبر استبدال عناصر نائبة بـ XML، ما يضمن توافقاً كاملاً مع Microsoft Word.

## ▶️ التشغيل محلياً

```bash
npm install
npm start          # http://localhost:3000
```
- الفورم: `http://localhost:3000/`
- لوحة الإدارة: `http://localhost:3000/admin.html`

## 🔌 نقاط الـ API

| الطريقة | المسار | الوظيفة |
|--------|--------|---------|
| POST   | `/api/submit`             | حفظ الطلب + توليد Word/Excel + بريد بالخلفية |
| GET    | `/api/submissions`        | قائمة الطلبات |
| GET    | `/api/submissions/:id`    | تفاصيل طلب |
| PUT    | `/api/submissions/:id`    | تعديل طلب وإعادة توليد ملفاته |
| DELETE | `/api/submissions/:id`    | حذف طلب |
| GET    | `/api/download/:id/word`  | تنزيل ملف Word |
| GET    | `/api/download/:id/excel` | تنزيل ملف Excel |
| GET    | `/api/email-test`         | تشخيص إعدادات البريد |

## 📧 البريد ومتغيّرات البيئة (Brevo)

يُرسَل البريد عبر **Brevo HTTP API** (يتجاوز حجب SMTP على الاستضافة). إن لم تُضبط المتغيّرات، يتخطّى النظام البريد ويُكمل عمله طبيعياً.

| المتغيّر | الوصف |
|---------|-------|
| `BREVO_API_KEY`  | مفتاح Brevo API (إلزامي للبريد) |
| `SENDER_EMAIL`   | إيميل المُرسِل المُفعَّل في Brevo (يقبل `EMAIL_USER` كبديل) |
| `RECEIVER_EMAIL` | المستلِم (اختياري؛ افتراضياً = المُرسِل) |
| `CC_EMAIL`       | نسخة كربونية (اختياري) |
| `PORT`           | المنفذ (اختياري؛ افتراضي 3000) |

للتشخيص: افتح `/api/email-test`.

## ☁️ النشر على Render

1. ارفع الملفات على GitHub عبر **Upload files** (اسحب الملفات المُستخرَجة — لا تحفظها من صفحة العرض)، مع مجلّد `assets/`.
2. أنشئ Web Service واربطه بالمستودع. Build: `npm install` — Start: `npm start`.
3. اضبط متغيّرات البيئة (أعلاه) ثم Deploy.

> ⚠️ مجلّد `data/` يُمسح مع كل إعادة نشر على خطة Render المجانية (نظام ملفّات مؤقّت). للحفظ الدائم استخدم **Render Disk** أو قاعدة بيانات.
